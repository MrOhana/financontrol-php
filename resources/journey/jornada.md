Jornada de aprendizado de Laravel

1- Copiei o especificacao.md do projeto de Java para o diretório resources/prompts

2- Consultei um link de boas práticas de arquitetura de Laravel para usar de referência no prompt. Link: https://www.google.com/search?q=boas+pr%C3%A1ticas+de+arquitetura+em+Laravel&oq=boas+pr%C3%A1ticas+de+arquitetura+em+Laravel

3- Após isso, eu precisava buscar alternativas ao Spring + Wicket que estavam na aplicação Java, assim eu escolhi ir de Laravel + Livewire + Alpine.js usando a própria documentação do Laravel: https://laravel.com/docs/12.x/frontend

4- Depois, tive que verificar ORMs para PHP, e descobri que o Laravel possui o Eloquent ORM que é muito similar ao JPA do Java. Link: https://laravel.com/docs/12.x/eloquent

5- Depois tive que pesquisar como o Laravel lidava com segurança, autorização, autenticação, hash de senhas, etc.

6- Hora de colocar o prompt no chat e ver o que o Antigravity pode me entregar. (O prompt original eu nunca altero. Se você quiser replicar a experiência, basta usar o mesmo)

7- Tiha esquecido de instalar o PHP e o Composer, mas não teve problema, o Antigravity o fez.

8- Hora de aceitar tudo, ter uma versão 1 e já subir no repositório. (se você for para o firts commit no projeto, você terá o código cru exatamente com o que o Antigravity gerou na primeira vez)

9- Salvei no diretório resources/journey as etapas do que o antigravity me guiou. O walkthrough.md é tudo o que foi feito na primeira interação e ele me explicou. O imp_plan_1.md foi o plano de execução da primeira interação.
